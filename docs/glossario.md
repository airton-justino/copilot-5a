# Glossário

Este glossário reúne os principais termos, siglas e definições utilizados nos projetos Salesforce da empresa 5A. O objetivo é garantir um vocabulário comum entre as áreas de negócio, atendimento e tecnologia.

---

## Termos Gerais

| Termo                  | Definição                                                                                                     |
|------------------------|---------------------------------------------------------------------------------------------------------------|
| **AS-IS**              | Descrição do processo atual, como ele funciona hoje, antes de qualquer mudança.                               |
| **TO-BE**              | Descrição do processo futuro, como ele deverá funcionar após a implementação das melhorias ou migrações.      |
| **Ponto em aberto**    | Informação, decisão ou definição ainda não confirmada, que precisa ser levantada e documentada.               |
| **Migração**           | Processo de transferência de dados e processos de um sistema para outro (neste caso, do Zendesk para o Salesforce). |
| **Integração**         | Conexão automatizada entre dois ou mais sistemas que permite a troca de informações em tempo real ou em lotes. |
| **SLA**                | Acordo de Nível de Serviço. Define os prazos máximos para resposta e resolução de casos de atendimento.       |

---

## Salesforce

| Termo                  | Definição                                                                                                     |
|------------------------|---------------------------------------------------------------------------------------------------------------|
| **Salesforce**         | Plataforma de CRM (gestão de relacionamento com clientes) utilizada pela empresa 5A.                          |
| **CRM**                | Customer Relationship Management. Sistema para gerenciar o relacionamento com clientes.                       |
| **Sales Cloud**        | Módulo do Salesforce focado em vendas, gestão de oportunidades e pipeline comercial.                          |
| **Service Cloud**      | Módulo do Salesforce focado em atendimento ao cliente e gestão de casos de suporte.                           |
| **Caso (Case)**        | Registro no Salesforce que representa uma solicitação, reclamação ou incidente de um cliente.                 |
| **Conta (Account)**    | Registro no Salesforce que representa uma empresa ou organização cliente.                                     |
| **Contato (Contact)**  | Registro no Salesforce que representa uma pessoa física vinculada a uma Conta.                                |
| **Lead**               | Registro no Salesforce que representa um potencial cliente ainda não qualificado.                             |
| **Oportunidade**       | Registro no Salesforce que representa uma negociação ou venda em andamento com um cliente.                    |
| **Objeto**             | Equivalente a uma tabela no banco de dados do Salesforce. Pode ser padrão (ex.: Caso, Conta) ou customizado.  |
| **Campo**              | Atributo de um objeto no Salesforce (equivalente a uma coluna em uma tabela).                                 |
| **Perfil**             | Conjunto de permissões que define o que um usuário pode ver e fazer no Salesforce.                            |
| **Fluxo (Flow)**       | Automação no Salesforce que executa ações com base em gatilhos e condições configuradas.                      |
| **Sandbox**            | Ambiente de testes no Salesforce, separado do ambiente de produção.                                           |
| **Org**                | Instância do Salesforce. Pode haver múltiplas orgs (ex.: produção e sandbox).                                 |

## Zendesk

| Termo                  | Definição                                                                                                     |
|------------------------|---------------------------------------------------------------------------------------------------------------|
| **Zendesk**            | Plataforma de atendimento ao cliente (helpdesk) utilizada atualmente pela empresa 5A.                         |
| **Ticket**             | Registro no Zendesk que representa uma solicitação ou chamado aberto por um cliente.                          |
| **Agente**             | Usuário do Zendesk responsável por atender e resolver os tickets dos clientes.                                |
| **Grupo**              | Conjunto de agentes no Zendesk que recebem tickets de um determinado tipo ou área.                            |
| **Organização**        | Registro no Zendesk que representa uma empresa cliente (equivalente à Conta no Salesforce).                   |
| **Usuário final**      | Cliente que abre tickets no Zendesk.                                                                          |
| **Gatilho (Trigger)**  | Regra no Zendesk que executa ações automaticamente quando certas condições são atendidas.                     |
| **Automação**          | Regra no Zendesk que executa ações com base em condições verificadas periodicamente.                          |
| **Vista (View)**       | Filtro de tickets no Zendesk que agrupa e exibe tickets com características específicas.                      |

| CPF     | Cadastro de Pessoa Física                        |
