# Backlog e Oportunidades

## Objetivo

Consolidar itens de backlog e oportunidades transversais citados no consolidado, sem tratar como decisoes finais.

## Backlog explicito no material

- Integracao com Twilio (Agendamento de Placas).
- Otimizacao da query Metabase para retirar clientes B2B direto na consulta (Agendamento de Placas).
- Gatilhar novo ticket linkado ao principal ou priorizar caso no Metabase (Pos Agendamento de Placas).
- Avaliar automacao para tratativas da Promove via API (Pos Agendamento de Placas).
- Definir SLA para fechamento automatico (Suporte Outsourcing).
- Transferir ticket automaticamente para outras filas (Reagendamentos Futuros).
- Preencher todas as informacoes no Cognito, sem dividir entre Cognito e Salesforce (Envio de fotos proprietario).
- Avaliar se PP Multi deveria ser viabilizado via produto (PP Multi).

## Oportunidades recorrentes

- Manter processo end-to-end no mesmo ticket/case quando houver viabilidade de API.
- Unificar formularios de entrada (tema recorrente em mais de um fluxo).
- Revisar estrategia de canais por processo.

## Pontos em aberto

- Priorizacao e responsaveis por cada item de backlog.
- Critérios de aceite para transicao de backlog para implementacao.
